# v0.1.0 Playable Alpha Release — Design Spec

## Goal
Ship NetHack Babel v0.1.0 as a GitHub Release with a prebuilt macOS ARM binary. A complete, playable NetHack 3.7 reimplementation.

## Scope

### IN v0.1.0
- Complete NetHack 3.7 engine (99.8% coverage, 4,103+ tests, 133K LOC)
- All 4 remaining wiring gaps closed
- TUI terminal interface (ratatui + crossterm)
- 5 languages (en, zh-CN, zh-TW, de partial, fr partial)
- Save/load with anti-savescum
- Leaderboard (JSON persistence)
- Wizard mode (-D flag)
- Per-game appearance shuffling
- 4-layer test pyramid + differential execution harness

### NOT in v0.1.0
- Non-macOS binaries (add in v0.1.1)
- Steam/itch.io distribution
- Graphical UI
- 149 hardcoded strings conversion (cosmetic)
- de/fr locale completion
- SSH multiplayer

## Phase 1: Close 4 Wiring Gaps

### 1a. Polymorph trap → polyself
- File: `crates/engine/src/turn.rs`
- When `TrapTriggered { trap_type: Polymorph }` event fires, call `polyself::polymorph_self()`
- Test: step on polymorph trap → form changes

### 1b. Vault guard spawn
- File: `crates/engine/src/turn.rs`
- In turn loop, check `vault::player_in_vault()` → spawn guard if not already present
- Test: enter vault room → guard appears

### 1c. Minion entity creation
- File: `crates/engine/src/religion.rs` or `turn.rs`
- When prayer succeeds with minion summon, create actual monster entity via `makemon`
- Test: pray → minion entity exists in world

### 1d. Appearance table → TUI display
- File: `crates/cli/src/main.rs` or `crates/tui/src/`
- When rendering item names, check `world.appearance_table` for unidentified items
- Test: unidentified potion shows shuffled color name

## Phase 2: Release Engineering

### 2a. Cargo.toml metadata
```toml
[package]
name = "nethack-babel"
version = "0.1.0"
edition = "2021"
description = "A modern Rust reimplementation of NetHack 3.7 with built-in multilingual support"
license = "NGPL"
homepage = "https://github.com/SatoshiNakamoto0126/nethack-babel"
repository = "https://github.com/SatoshiNakamoto0126/nethack-babel"
readme = "README.md"
keywords = ["nethack", "roguelike", "game", "tui", "i18n"]
categories = ["games"]
```

### 2b. GitHub Actions CI
- File: `.github/workflows/ci.yml`
- Trigger: push to master, pull requests
- Jobs: `cargo check`, `cargo test --workspace`, `cargo clippy`
- Platform: macOS ARM (self-hosted or macos-14)

### 2c. Release workflow
- File: `.github/workflows/release.yml`
- Trigger: tag push `v*`
- Build: `cargo build --release --target aarch64-apple-darwin`
- Package: strip binary, tar.gz with data/ directory
- Upload: GitHub Release with binary + changelog

### 2d. CHANGELOG.md
- Document all major features for v0.1.0
- Follow Keep a Changelog format

### 2e. Install script
- `install.sh`: download binary, extract, add to PATH

## Phase 3: Verification

1. `cargo test --workspace` — 0 failures
2. Fresh game: `cargo run -- --data-dir data` → character creation works
3. Gameplay: move, fight, eat, quaff, read, zap — all functional
4. Save/load: save mid-game → restart → load continues
5. Wizard mode: `-D` flag → Ctrl+W wish → item appears
6. Death: HP reaches 0 → tombstone → leaderboard entry
7. Chinese: `--language zh_CN` → all text in Chinese
8. Binary: `cargo build --release` → `target/release/nethack-babel` runs standalone
